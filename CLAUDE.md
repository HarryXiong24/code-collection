# code-collection

Personal learning and interview prep repository covering full-stack development, algorithms, and system design across multiple languages.

## Structure

| Directory                     | Purpose                                                                                                                |
| ----------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| `Data Structure & Algorithm/` | Graph, Greedy, DP, algorithm templates (old + new versions)                                                            |
| `FrontEnd/`                   | FE interview questions, Vue 2/3 internals, React, TypeScript challenges, Promise/Debounce/EventEmitter implementations |
| `Backend/`                    | Go (primary), Node.js, Python; includes 58 tricky Go edge cases                                                        |
| `Coding/`                     | Sliding Window, Two Pointers, Stack, Intervals — Go implementations                                                    |
| `Database/`                   | LeetCode SQL and general SQL problem sets                                                                              |
| `Micro Project/`              | System design patterns: LRU Cache, Circuit Breaker, Rate Limiter, Bounded Queue, SSE — in both TypeScript and Go       |
| `iOS/`                        | Swift and Objective-C data structure implementations                                                                   |
| `Other/`                      | Staged review materials (Stage 1–11)                                                                                   |
| `Tests/`                      | Scratch files (test.go, test.js, test.ts, test.py)                                                                     |

## Languages

- TypeScript / JavaScript — frontend focus (72% of files)
- Go — backend and algorithm practice (16.5%)
- Python — data structures and notes (10%)
- Swift / Objective-C — mobile (1%)

## Conventions

- LeetCode problems use a numbered prefix: `200. Number of Islands`
- Hard or important problems are marked with `@`: `@376. Wiggle Subsequence`
- Core patterns (LRU Cache, Rate Limiter, etc.) are implemented in both TypeScript and Go for cross-language comparison
- Staged review folders (`Stage 1` through `Stage 11`) support spaced repetition practice
