# Code Collection

> A personal knowledge base and coding practice repository covering data structures & algorithms, full-stack development, system design, and AI.

[![License: MIT](https://img.shields.io/badge/License-MIT-blue.svg)](LICENSE)

## Repository Structure

```
code-collection/
├── AI & Agent/                   # AI/ML and agent experiments
├── Data Structure & Algorithm/   # Algorithms and coding practice
├── Full Stack Develop/           # Frontend, backend, database, mobile
├── In Progress/                  # Work in progress — not yet categorized
├── System Design/                # Design patterns as runnable micro projects
└── Temp/                         # Throwaway experiments
```

| Directory | Description |
|-----------|-------------|
| **AI & Agent** | AI/ML and agent exploration — NanoGPT, micrograd MLP, MCP, RAG, ReAct |
| **Data Structure & Algorithm** | Algorithm problems, reference implementations, templates, and spaced-repetition practice |
| **Full Stack Develop** | Applied engineering across the stack — frontend internals, Golang/Node/Python backends, SQL, iOS |
| **In Progress** | **Projects currently being worked on, not yet filed into any category above.** A queue, not a home — items graduate out once they settle |
| **System Design** | Design patterns built as working micro projects — Rate Limiter, Circuit Breaker, LRU Cache, Bounded Blocking Queue, SSE |
| **Temp** | Throwaway experiments and scratch files. Unlike **In Progress**, nothing here is meant to last |

## Conventions

- Numbered problems correspond to [LeetCode](https://leetcode.com/) IDs: `200. Number of Islands`
- Hard or important problems are prefixed with `@`: `@376. Wiggle Subsequence`
- The same problem appearing in several places is intentional — solving it again from a different angle is the method, not redundancy

## 目录概览

| 目录 | 说明 |
|------|------|
| **AI & Agent** | AI 与 Agent 探索 —— NanoGPT、micrograd MLP、MCP、RAG、ReAct |
| **Data Structure & Algorithm** | 算法题目、参考实现、模板与间隔重复练习 |
| **Full Stack Develop** | 全栈工程实践 —— 前端原理、Golang/Node/Python 后端、SQL、iOS |
| **In Progress** | **当前正在进行中、尚未归类到上述任何目录的项目。** 它是队列而非归宿 —— 项目稳定后应毕业到对应分类 |
| **System Design** | 以可运行微项目实现的设计模式 —— 限流器、熔断器、LRU 缓存、有界阻塞队列、SSE |
| **Temp** | 临时试验与草稿文件。与 **In Progress** 不同，这里的东西不打算长期保留 |

## 说明

- 带有序号的算法题目对应 [LeetCode](https://leetcode.com/) 题号，`@` 前缀表示重点或困难题
- 同一题目在多处出现是刻意为之 —— 换个角度把同一题再解一遍正是方法本身，而非冗余
